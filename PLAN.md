## 🗺️ Архитектура проекта

```
/photodesk/
├── photodesk.html          CRM-дашборд (основной рабочий инструмент)
├── booking.html            Публичная форма бронирования для клиентов
├── import.html             Утилита одноразового импорта в Firestore
├── PLAN.md                 ← этот файл
└── grok_cleaner/
    ├── app.py              Desktop-приложение (Tkinter) для AI-обработки фото
    ├── requirements.txt    requests, Pillow, rawpy
    └── run.sh              Запуск одной командой / двойным кликом
```

**Два независимых продукта:**

| Продукт | Назначение | Стек | Git |
|---|---|---|---|
| **PhotoDesk CRM** | Управление заказами, клиентами, нкасим | HTML/CSS/JS · Firebase Firestore | github.com/edmuntas/photodesk (GitHub Pages) |
| **Grok Batch Cleaner** | AI-очистка и обработка фото недвижимости | Python · Tkinter · xAI API | github.com/edmontmac16max/photodesk-grok-cleaner |

### P3 — Архив удалённых записей 🟡 Средний приоритет

**Проблема:** Сейчас удаление нкасим, клиентов и משрדим — **постоянное и необратимое**.
**Решение:** Soft-delete — записи помечаются как удалённые и переносятся в архив.

**Что сделать:**
- [ ] Добавить в store: `store.archive = { listings: [], customers: [], companies: [] }`
- [ ] Вместо `filter(x => x.id !== id)` — перемещать в `store.archive.X` с полем `deletedAt`
- [ ] Новая вкладка "🗄️ ארכיון" в sidebar (внизу, под Настройками)
- [ ] Страница архива: 3 таба — נכסים / לקוחות / משרדים
- [ ] В каждой строке: кнопка "שחזר" (восстановить) и "מחק לצמיתות" (удалить навсегда)
- [ ] Автоочистка архива: записи старше 90 дней удаляются навсегда
- [ ] Счётчик на иконке архива — сколько записей в архиве
- [ ] Фильтры: по дате удаления, по типу, поиск по адресу/имени

**Технические детали:**
- `deleteListingInline(id)` → `archiveListing(id)` — перемещает в `store.archive.listings`
- `deleteCompany(id)` → `archiveCompany(id)`
- `deleteCustomer()` → `archiveCustomer()`
- `saveStore()` уже сохраняет весь объект store — архив сохранится автоматически
