# Флоу клиента (агента по недвижимости)
*[[01-moc/PhotoDesk MOC|← MOC]]*

---

## Шаг 1 — Бронирование (booking.html)

```
Агент получает ссылку на форму:
https://edmuntas.github.io/photodesk/booking.html

Шаг 1/4: Выбор клиента
  → Я уже клиент (поиск по имени/телефону)
  → Новый клиент (заполнить данные)

Шаг 2/4: Данные объекта
  → Адрес, город, район
  → Комнаты, санузлы, этаж, площадь, год
  → Заметки

Шаг 3/4: Выбор пакета
  → Basic ₪800 / Premium ₪1,200 / Unlimited ₪2,000
  (Пакеты синхронизированы с CRM)

Шаг 4/4: Дата и гибкость
  → Конкретная дата / Диапазон недели / Как можно скорее

✅ Отправка → email фотографу + запись в Firestore
```

---

## Шаг 2 — Квота (quote.html)

```
Фотограф создаёт квоту → отправляет ссылку клиенту:
https://edmuntas.github.io/photodesk/quote.html?id=LISTING_ID

Клиент видит:
  → Детали объекта
  → Интерактивный калькулятор цены по типу объекта
  → Пакет и услуги
  → Контакт фотографа

Клиент нажимает "Подтверждаю" →
  → Listing статус → "Запланирован"
  → Уведомление фотографу

Экспорт PDF / Поделиться в WhatsApp
```

---

## Шаг 3 — Подготовка (prepare.html)

```
Фотограф отправляет ссылку:
https://edmuntas.github.io/photodesk/prepare.html

Гайд по подготовке объекта:
  → Что убрать
  → Как подготовить комнаты
  → Время прихода фотографа
  → Что будет происходить
```

---

## Шаг 4 — Доставка (delivery.html)

```
Фотограф загружает фото → отправляет ссылку:
https://edmuntas.github.io/photodesk/delivery.html?id=LISTING_ID

Клиент видит:
  → Галерея готовых фото
  → Скачать отдельное фото
  → Скачать все (ZIP)
  → Видео (если есть)
  → Планы этажей (если есть)
```

## Точки контакта (Touchpoints)

| Этап | Инструмент | Кто действует |
|---|---|---|
| Запрос на съёмку | booking.html | Клиент |
| Подтверждение заявки | CRM → email | Фотограф |
| Гайд подготовки | prepare.html | Клиент |
| Квота | quote.html | Клиент подтверждает |
| День съёмки | — | Фотограф на месте |
| Доставка | delivery.html | Клиент скачивает |
| Оплата | Счёт (вне системы) | Клиент |
