## 🔴 Критичные пробелы

### 1. Нет реального хранилища файлов
**Проблема:** Фотограф не может загрузить фото в систему.
**Риск:** Основная ценность продукта — доставка фото — не работает end-to-end.
**Решение:** Firebase Storage + bulk upload + progress tracking
**Сложность:** Medium | **Приоритет:** Срочно

---

### 2. Клиент не получает уведомлений
**Проблема:** Клиент не знает, когда съёмка назначена, когда фото готовы.
**Риск:** Агенты звонят фотографу сами → лишняя нагрузка.
**Решение:**
- Email триггеры на смену статусов (Firebase Functions + SendGrid/Mailgun)
- WhatsApp Business API для израильского рынка
**Сложность:** Medium | **Приоритет:** Высокий

---

### 3. Нет синхронизации CRM ↔ Grok Cleaner
**Проблема:** Фотограф обрабатывает фото в Grok → вручную загружает в CRM.
**Риск:** 2 разных системы = дублирование работы + ошибки.
**Решение:** После обработки в Grok Cleaner — кнопка "Отправить в CRM" → Firebase Storage upload → listing.photos обновляется.
**Сложность:** Medium | **Приоритет:** Средний

## 🟡 Важные возможности

### 4. Автоматические статусы
**Возможность:** Часть переходов статусов можно автоматизировать:
- Клиент подтвердил квоту → "Запланирован" (уже есть)
- Фото загружены → "В обработке"
- delivery.html открыт клиентом → "Доставлен"
- Прошло 30 дней без оплаты → уведомление

### 5. Финансовая аналитика
**Возможность:** Текущий CRM хранит все данные для отчётности.
- Выручка по месяцам
- Топ-клиенты по объёму
- Загруженность по дням недели
- Конверсия заявок → сделки
**Сложность:** Low (данные уже есть) | **Ценность:** Высокая

### 6. WhatsApp-первый подход
**Возможность:** Израильский рынок — WhatsApp является основным каналом коммуникации.
- Шаблонные сообщения для каждого этапа
- Кнопка "Отправить в WhatsApp" уже есть частично
- Можно интегрировать WhatsApp Business API через 360dialog

## 🟢 Долгосрочные возможности

### 7. Мультипользовательский режим
**Текущее:** Один фотограф = один аккаунт.
**Возможность:** Небольшие студии (2-5 фотографов), ассистенты.
**Что нужно:** Roles & permissions в Firestore, multi-tenant архитектура.
**Сложность:** High

### 8. Client Portal (личный кабинет)
**Возможность:** Агент заходит под своим логином → видит только свои объекты, историю, счета.
**Ценность:** Снижает количество звонков "где мои фото?".
**Сложность:** High

### 9. Интеграция с израильскими порталами недвижимости
**Возможность:** Автоматическая публикация готовых фото на:
- Yad2 (יד 2)
- Madlan (מדלן)
- Homely
**Сложность:** High (нет открытого API у порталов)

### 10. AI-улучшения Grok Cleaner
**Возможность:**
- Авто-определение типа комнаты по содержимому изображения (не по имени файла)
- Пресеты "Израильский стиль" / "Скандинавский" / "Минимализм"
- Пакетная обработка из CRM: "Обработать все фото для этого Listing"

## Технические риски

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Firebase costs scaling | Средняя | Средний | Мониторинг reads/writes |
| xAI API downtime | Средняя | Высокий | Fallback на альтернативный API |
| Single-file HTML сложность | Высокая | Средний | Модуляризация при >10K строк |
| GitHub Pages лимиты | Низкая | Низкий | Переход на Firebase Hosting |
| Grok API price changes | Средняя | Высокий | Тестировать Anthropic/OpenAI как альтернативу |
