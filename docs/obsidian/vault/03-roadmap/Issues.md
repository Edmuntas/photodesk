## 🔴 Критичные (production blockers)

### B01 — Firebase Storage не подключён
- **Файл:** photodesk.html (вкладка Медиа в Listing)
- **Описание:** UI для загрузки фото/видео/планов существует, но файлы не сохраняются. Storage bucket не инициализирован.
- **Влияние:** Основная функция доставки фото не работает.
- **Fix:** Инициализировать Firebase Storage, добавить uploadBytes() + getDownloadURL()

### B03 — Нет real-time обновлений
- **Файл:** photodesk.html
- **Описание:** Данные загружаются один раз при входе. Новые заявки и изменения от клиентов не появляются без ручного обновления страницы.
- **Влияние:** Можно пропустить новую заявку.
- **Fix:** Добавить `onSnapshot()` listener для orders/bookings collection

---

## 🟡 Важные

### B04 — Нет бейджа на Orders
- **Описание:** В сайдбаре нет счётчика непросмотренных заявок.
- **Fix:** Listener на bookings, где status === 'new' → отображать number badge

### B05 — Календарь без реальных данных
- **Описание:** Месячный вид частично показывает статические данные. Не все объекты с датами отображаются.
- **Fix:** Query listings по shootDate, render markers на соответствующих датах

### B06 — prompt() вместо модальных окон
- **Описание:** Редактирование полей использует нативный `prompt()` браузера. Это ломает UX, не поддерживает иврит корректно, и выглядит непрофессионально.
- **Fix:** Создать переиспользуемый компонент Modal с input/textarea

### B07 — Нет toast-уведомлений
- **Описание:** Операции (сохранение, удаление) не дают визуального подтверждения.
- **Fix:** Добавить lightweight toast system (CSS + JS, без библиотек)

## 🟢 Незначительные

### B08 — phone.replace() crash
- **Статус:** ✅ Исправлен (коммит 17151ec)
- **Описание:** Краш при phone === null при форматировании

### B09 — Race condition в loadListings
- **Статус:** ✅ Исправлен (коммит 17151ec)
- **Описание:** Параллельные вызовы перезаписывали данные

### B10 — Archive auto-cleanup threshold
- **Описание:** 90-дневный лимит hard-coded. Нет настройки.
- **Важность:** Low
