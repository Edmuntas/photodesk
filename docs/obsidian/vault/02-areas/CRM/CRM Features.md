## ✅ Реализованные фичи

### 1. Аутентификация
- **Тип:** Backend / UI
- **Статус:** Done
- **Описание:** Firebase Auth с email/password. Оверлей логина при старте, сессия сохраняется. Кнопка выхода в сайдбаре.
- **Input:** email + пароль
- **Output:** Доступ к CRM
- **Зависимости:** Firebase Auth SDK
- **Сложность:** Low
- **Важность:** Core

---

### 2. Listings (Объекты недвижимости)
- **Тип:** UI / Backend
- **Статус:** Done
- **Описание:** Главная таблица всех объектов. Карточки/таблица с фильтрацией по статусу. Клик → выдвигающаяся панель с 7 табами.
- **Input:** Данные объекта (адрес, город, тип, площадь, комнаты, клиент, пакет, дата)
- **Output:** Запись в Firestore; отображение в таблице
- **Статусы объекта:** חדש (новый) / ממתין (ожидает) / מתוכנן (запланирован) / צולם (снят) / בעיבוד (в обработке) / נמסר (доставлен) / בוטל (отменён)
- **Зависимости:** Firestore `listings` collection
- **Сложность:** Medium
- **Важность:** Core

#### Детальная панель объекта (7 табов):
| Таб | Содержание |
|---|---|
| פרטים (Детали) | Основная информация объекта |
| מדיה (Медиа) | Загрузка фото/видео/планов |
| תשלום (Оплата) | Статус оплаты, сумма |
| הערות (Заметки) | Свободный текст |
| שיתוף (Шаринг) | Ссылки для клиента |
| היסטוריה (История) | Лог изменений |
| קבצים (Файлы) | Прикреплённые файлы |

### 3. Dashboard (Дашборд)
- **Тип:** UI
- **Статус:** Done
- **Описание:** Главная страница с KPI-картами, пайплайном, неоплаченными счетами, мини-календарём, баннером сегодняшних съёмок.
- **Компоненты:**
  - Статистические карты (кликабельные фильтры): Всего объектов / В работе / Нет оплаты / Доставлено
  - Канбан-пайплайн (горизонтальный): 5 колонок по статусу
  - Список неоплаченных с суммами
  - Предстоящие съёмки (следующие 7 дней)
  - Баннер "Сегодняшняя съёмка"
- **Зависимости:** Firestore, store object
- **Сложность:** Medium
- **Важность:** Core

---

### 4. Customers (Клиенты)
- **Тип:** UI / Backend
- **Статус:** Done
- **Описание:** Двухуровневая иерархия: Компании (офисы) → Агенты. CRUD для обоих уровней. Функция слияния клиентов (merge).
- **Input:** Имя, телефон, email, компания, тип
- **Output:** Записи в `customers` + `companies`
- **Функции:**
  - Создать/редактировать/удалить компанию
  - Создать/редактировать/удалить агента
  - Привязать агента к компании
  - Merge: объединить дублирующиеся записи
  - Поиск по имени/телефону/email
  - Клик → панель с историей заказов агента
- **Зависимости:** `customers`, `companies` collections
- **Сложность:** Medium
- **Важность:** Core

---

### 5. Orders (Заявки)
- **Тип:** UI / Backend
- **Статус:** Done
- **Описание:** Входящие заявки из `booking.html`. Статус: новые / одобрены / отклонены. Одобрение создаёт Listing.
- **Input:** Данные из booking.html
- **Output:** Новый Listing при одобрении
- **Проблема:** Нет real-time слушателя — нужен ручной refresh ⚠️
- **Зависимости:** `bookings` / `orders` collection
- **Сложность:** Medium
- **Важность:** Core

---

### 6. Calendar (Календарь)
- **Тип:** UI
- **Статус:** Partial ⚠️
- **Описание:** Месячный вид с маркерами съёмок. Клик на дату показывает объекты.
- **Проблема:** Частично статические данные, нет синхронизации с реальными датами из Firestore
- **Сложность:** Medium
- **Важность:** Supporting

### 7. Packages (Пакеты)
- **Тип:** UI / Backend
- **Статус:** Done
- **Описание:** Управление пакетами услуг. Данные синхронизируются в `booking.html` и `quote.html`.
- **Пакеты по умолчанию:**

- **Зависимости:** `store` collection
- **Сложность:** Low
- **Важность:** Supporting

### 13. Quote Generator (Генератор кавычек)
- **Тип:** UI / Workflow
- **Статус:** Done
- **Файл:** `quote.html`
- **Описание:** Публичная страница расчёта стоимости по ID объекта. Интерактивный калькулятор с 15 типами недвижимости и множителями. Клиент подтверждает → статус listing меняется на "Запланирован".
- **Input:** `?id=LISTING_ID` в URL
- **Output:** PDF квота, WhatsApp-шаринг, подтверждение клиента
- **Зависимости:** Firestore listing read, PDF export
- **Сложность:** Medium
- **Важность:** Core

---

### 14. Booking Form (Форма записи)
- **Тип:** UI / Workflow
- **Статус:** Done
- **Файл:** `booking.html`
- **Описание:** 4-шаговый визард для клиентов. Шаги: выбор клиента → детали объекта → пакет → дата.
- **Input:** Данные клиента + объекта
- **Output:** Запись в `bookings`, email-уведомление на edmontdoron@gmail.com
- **Зависимости:** Firestore, formsubmit.co
- **Сложность:** Medium
- **Важность:** Core

## 🚧 Частично реализованные

### F-P1: Real-time Firestore Listeners
- **Статус:** In Progress
- **Проблема:** Данные обновляются только при ручном refresh. Нужны `onSnapshot()` listeners.
- **Приоритет:** 🔴 Критично

### F-P2: Firebase Storage (загрузка файлов)
- **Статус:** In Progress
- **Проблема:** UI для загрузки фото/видео/планов есть, но нет реального сохранения в Firebase Storage.
- **Приоритет:** 🔴 Критично

### F-P3: Modal Editors
- **Статус:** In Progress
- **Проблема:** Для редактирования используются нативные `prompt()` диалоги браузера. Нужны кастомные модальные окна.
- **Приоритет:** 🟡 Важно

### F-P4: Calendar with Real Data
- **Статус:** In Progress
- **Проблема:** Календарь показывает статические данные. Нужна синхронизация с реальными датами из Firestore.
- **Приоритет:** 🟡 Важно

### F-P5: Badge Counter (Orders)
- **Тип:** UI
- **Статус:** In Progress
- **Описание:** Счётчик непросмотренных заявок на иконке "Заказы" в сайдбаре
- **Приоритет:** 🟡 Важно
