## Структура PARA

| Папка | Назначение |
|-------|-----------|
| `00-inbox/` | Новые необработанные заметки — сортировать отсюда |
| `02-areas/` | Постоянные зоны ответственности (бизнес, финансы, клиенты) |
| `topics/` | Тематические заметки — концепции, технологии, процессы |
| `sources/` | Источники — статьи, книги, видео, ссылки |

## Правила работы с vault

### Создание заметок
- **Новые темы** → `topics/<название>.md`
- **Источники** (статьи, книги, ссылки) → `sources/<название>.md`
- **Входящее** (необработанное) → `00-inbox/<название>.md`
- **Зоны ответственности** → `02-areas/<название>.md`

### Ссылки
- Всегда используй **[[вики-ссылки]]** для связей между заметками
- Никогда не используй обычные markdown-ссылки для внутренних заметок
- При создании новой заметки — добавляй ссылки на связанные существующие заметки

### Форматирование
- Все файлы — `.md` (Markdown)
- Первая строка заметки — заголовок `# Название`
- Добавляй YAML frontmatter если нужны теги или дата:
  ```yaml
  ---
  tags: [тег1, тег2]
  date: YYYY-MM-DD
  ---
  ```

### Чего не делать
- Не создавай папки вне структуры PARA без явного запроса
- Не удаляй заметки — перемещай в `00-inbox/` если не знаешь куда
- Не изменяй структуру существующих заметок без запроса
